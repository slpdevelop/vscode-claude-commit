🅰️  ANTHROPIC API (если есть API ключ)

   1. Получите ключ: https://console.anthropic.com/
   2. VSCode Settings (Ctrl+,) → "Claude Commit"
   3. Provider = anthropic
   4. Api Key = sk-ant-...

🅱️  AWS BEDROCK (БЕЗ API ключа!) ← ВАША СИТУАЦИЯ

   1. Настройте AWS:
      
      aws configure
      
      Введите Access Key, Secret Key, region (us-east-1)

   2. Включите Claude в Bedrock Console:
      https://console.aws.amazon.com/bedrock/
      → Model access → Request access для Claude

   3. VSCode Settings (Ctrl+,) → "Claude Commit"
      Provider = bedrock
      Aws Region = us-east-1
      Aws Profile = (пусто для default)

   📖 Детали: см. BEDROCK_SETUP.md

Так как у вас НЕТ API ключа Anthropic, используйте AWS Bedrock:

✅ 1. Установите расширение (см. INSTALL.md)

✅ 2. Настройте AWS credentials:
      
      aws configure

✅ 3. Включите Claude в AWS Bedrock Console:
      
      https://console.aws.amazon.com/bedrock/
      → us-east-1 регион
      → Model access
      → Request model access для Claude

✅ 4. В VSCode настройте:
      
      Provider: bedrock
      Aws Region: us-east-1

✅ 5. Попробуйте на любом git repo!

═══════════════════════════════════════════════════════════════════
                TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════

❌ "AccessDeniedException" (Bedrock)
   → Добавьте IAM permission для bedrock:InvokeModel
   → См. BEDROCK_SETUP.md секция IAM Policy

❌ "Model not found" (Bedrock)
   → Включите модель в Bedrock Console
   → Model access → Request model access

❌ "Unable to locate credentials" (Bedrock)
   → Выполните: aws configure

❌ "API key not configured" (Anthropic)
   → Используйте Bedrock вместо Anthropic!
   → Смените Provider на "bedrock"

Больше решений: BEDROCK_SETUP.md → Troubleshooting
